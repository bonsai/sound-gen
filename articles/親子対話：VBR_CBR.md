# 親子対話：VBRとCBR

父「また新しい子だ。VBR？CBR？MP3のやつか？」

娘「**Variable BitRate と Constant BitRate**——音声コーデックのビットレート制御方式。料理で言うと、**CBRが『全食材を同じサイズに切る』（無駄あり）で、VBRが『食材に応じて大きさを変える』（効率的）**」

父：「MP3のVBRとか聞いたことあるな。でもニューラルコーデックでも同じ概念が使えるのか」

娘：「そう。**EnCodecやMimiのようなニューラルコーデックは基本CBR**——すべてのフレームに同じ数のトークンを割り当てる。でも音声は**無音部分より有声部分の方が情報量が多い**。CBRは無音にも同じビットを割いて無駄が生じる」

父：「VBRならどうなるんだ？」

娘：「**複雑な部分（有声・高周波）には多めのトークン、単純な部分（無音・定常ノイズ）には少なめのトークン**を割り当てる。結果、**同じビットレートならVBRの方が高品質**。**同じ品質ならVBRの方が低ビットレート**」

父：「じゃあVBRの方がいいに決まってるじゃないか。なぜCBRも使われるんだ？」

娘：「**予測可能性と遅延の問題**。CBRはフレームごとのビット数が一定なので、転送速度やバッファ管理が簡単。VBRはフレームごとにビット数が変わるので、**リアルタイム通信ではバッファ制御が難しくなる**。VoIPやビデオ会議ではCBRが好まれる理由」

父：「ニューラルコーデックの世界ではどうなんだ？」

娘：「最近の研究（Interspeech 2025）では**VRVQ（Variable Bitrate RVQ）** という手法が提案された。**フレームごとにRVQの層数を動的に変える**ことで、CBRのRVQより同じビットレートで高品質を達成。ノイズが多い環境でも頑健」

父：「VBRにできれば効率的だが、リアルタイム性が必要ならCBR——適材適所か」

娘：「大正解😊 そして**Speech LMの文脈ではVBRが特に重要**。言語モデルはトークン列の長さに直接計算量が比例する。**VBRで無駄なトークンを減らせば、LLMの推論も高速化・省メモリ化**できる——**音声AI全体の効率に直結する地味すぎる重要技術**だよ」

父：「なるほど。通信でもAIでも、無駄を省くのが一番の近道ってわけか」
