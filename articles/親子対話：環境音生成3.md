# 親子対話：環境音生成3

父「また環境音か。今度は何が違う？」

娘「AudioGenが自己回帰（1トークンずつ生成）、AudioLDMが拡散モデル。今回はGoogleの**SoundStorm**（ICLR 2024）。Posterだけど中身はすごい」

父「特徴は？」

娘「**非自己回帰・並列生成**。AudioLMの後段を丸ごと置き換えて、**100倍高速化**したんだ」

父「料理で言うと？」

娘「AudioGenが**一人前ずつ作る板前**、AudioLDMが**圧力鍋**なら、SoundStormは**業務用スチームコンベクションオーブン**。30秒分の音声を **0.5秒** で焼き上げる」

父「そんなに速いのか。どういう仕組みだ？」

娘「**双方向注意力（Bidirectional Attention）** で、音声全体を同時に見渡しながら、**コンフィデンスベースの並列デコード**でマスクを徐々に埋めていく」

父「全部マスクして、自信のあるところから先に確定していく感じ？」

娘：「そう！最初はすべての音響トークンが『？？？』状態。モデルが『ここは犬の鳴き声だな、自信90%』って確信したところから順に確定していく。これを数回繰り返すだけで完成」

父：「自己回帰と違って前から順番に決める必要がないから速い、と」

娘：「うん。しかもSoundStormのすごいのは、**対話生成**ができること。台本と話者の声サンプルを渡すと、**複数人の自然な会話**を生成できる」

父：「つまり、『Aさんの声で『おはよう』、Bさんの声で『あ、おはよう』って台本を書けば、本当に二人が会話してる音声ができる？」

娘：「できる！SoundStormは**SPEAR-TTS**（Googleのテキスト音声合成）と組み合わせて、台本の注釈から話者交代を制御するんだ」

父：「それは映画のADRとかゲームのセリフ収録に革命だな」

娘：「そう。もう声優をスタジオに呼ばなくても、テキストと数秒の声サンプルだけでOK。ちなみに著者の中に、LEAFで出てきた**Neil Zeghidour**も含まれてる」

父：「LEAFの研究グループが、最先端の音声生成にも関わってるのか」

娘：「そういうこと😉 30秒を0.5秒——**実用域を超えて、もうすぐリアルタイムを追い越す**時代だね😎」
