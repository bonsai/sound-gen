# 親子対話：音声S2Sの宿命【前編】——速さか、賢さか

父「また新しい子が出た。S2S？しかもカスケードってなんだ」

娘「**Speech-to-Speech——音声から音声への直接生成**。料理で言うと、**S2Sは『食材（音声）を見て一品作る一貫厨房』で、カスケードは『食材を仕分け→レシピ検索→調理』の分業制**」

父「なるほど。音声入力をそのまま音声出力に変換する、ってことか」

娘「そう。でもS2Sは**単一モデルですべてをやる**。一方の**カスケード方式**は3段階——ASR（音声認識）でテキスト化→LLMで思考→TTS（音声合成）で読み上げ。**S2Sは速いが浅く、カスケードは遅いが深い**」

父：「トレードオフか。どのくらい違うんだ？」

娘「カスケードは**遅延約2.1秒**——ASRが発話終了を待ってから動き出すから。S2Sは**ユーザーが話し終わる前に返事を始められる**。でもS2Sの内部表現は音声トークン（韻律・感情・リズムも含む）なので、LLMの膨大な知識を詰め込む余裕がない」

父：「じゃあS2Sって賢くないのか？」

娘：「**S2SでもLLMを組み込んだ例はある**。**Moshi**（Kyutai, 2024）は**自己回帰的に音声トークンを生成**しつつ、内部にテキストの『内言（inner monologue）』ストリームを持つ。音声を生成しながら頭の中で言語処理も同時にやる——料理で言うと、**味見しながら次の味付けを考える**感じ」

父：「面白いな。でもそれでも限界があると？」

娘：「そこに登場したのが**Sakana AIのKAME**（ICASSP 2026）。**タンデム方式**——表でS2S（Moshiベース）が即座に喋り始め、**裏でLLMが非同期に推論**して、『オラクル信号』として結果を注入する。**『考えてから話す』ではなく『話しながら考える』**を実現した」

父：「S2Sがフロントの料理人で、裏でシェフ（LLM）が別の料理を仕込んでる感じか」

娘：「大正解😊 フロントは**80ミリ秒サイクル**で即応答。裏のLLMはGPT-4.1でもClaudeでもGeminiでも**差し替え自由**。オラクルが更新されるたびにフロントが**話の途中で軌道修正**する——人間の会話と同じ」

父：「つまり、S2Sは『速い』『カスケードは『賢い』。そしてKAMEは『速くて賢い』を両立した新しい解か」

娘：「そういうこと😎 次回はこのKAMEの仕組みを深掘りするよ——**音声AIの『話しながら考える』時代**、ここから始まる」
