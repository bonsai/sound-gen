# 親子対話：DAC

父「DAC？また新しいコーデックか」

娘「**Descript Audio Codec**（.dac）——Descript社が2023年に発表、**Improved RVQGAN** という論文。料理で言うと、**『真空パックの技術を一気に進化させた次世代保存パック』**——44.1kHzの高音質オーディオを **たった8kbpsに圧縮（約90倍圧縮）** しながら、驚異的な再構成品質を実現」

父：「EnCodecよりすごいのか？」

娘：「**桁違いにすごい**。EnCodecが24kbps / 48kHz / 32倍圧縮なのに対し、DACは **8kbps / 44.1kHz / 90倍圧縮**。しかも単一のユニバーサルモデルで **音声・環境音・音楽すべてのドメインをカバー** する。MUSHRA主観評価でも EnCodec を明確に上回った」

父：「どうやってそんな高圧縮を実現したんだ？」

娘：「3つの革新。(1) **画像領域のImproved VQGAN技術を音声に導入**——VQのコードブック学習を安定化。(2) **改良された敵対的損失＋マルチスケールSTFT損失**——高忠実度再構築。(3) **大規模なRVQ**——16の階層的コードブックで 2^160 通りの組み合わせから最適な量子化を選択。ちなみにコードブックサイズは 1024」

父：「16層も！でもそれで8kbpsに収まるのか？」

娘：「フレームレートが 86Hz（EnCodecの75Hzよりやや高い）で、1フレームあたり16トークン×16bit（コードブックインデックス）= 256bit。86 × 256 = 約22kbps……に見えるが、**VRVQのように可変ではなく、固定の8kbpsモデル** を別に訓練している。つまりコードブック数を減らした専用モデル」

父：「つまり、用途に応じてトレードオフを調整したモデル群を提供してるのか」

娘：「その通り！16kHz / 24kHz / 44.1kHz のサンプリングレート別に重みを公開。**pip install descript-audio-codec で即インストール可能**。Hugging Face Transformersにも統合済み。そしてDACは **AudioLM / MusicGen / 音声LLMのトークナイザーとして広く使われている** — 今やニューラルオーディオコーデックのデファクトスタンダードの一つだよ😊」

父：「なるほど。EnCodecより進んでるけど、Mimi（Kyutai）やSNACなんかも出てきて、コーデック戦国時代って感じだな」

娘：「まさに！DAC, EnCodec, SoundStream, Mimi, SNAC — **コーデック戦国時代の真っただ中**。どれが音声LLMの標準トークナイザーになるかはまだ決着ついてない。DACは **高品質・高圧縮・汎用性の三拍子揃った最右翼** 🔥」
