# 親子対話：音声トークン

父「また新しい子が出た。音声トークン？」

娘「**音声を離散的な記号に変換したもの**。料理で言うと、**巨大な食材（波形）をAIが扱える一口大に切り分ける**イメージ。切り方（コードブックのサイズ）によって再現度が変わる」

父「トークンって、LLMで言う『言葉の断片』みたいなものか。でも音声にもトークンがあるのか」

娘：「音声LLMの時代になって一気に重要になった概念。**EnCodecやMimiのようなニューラルコーデック**で音声をトークン列に変換すると、**言語モデル（LLM）が音声を『言葉』と同じように処理できる**」

父：「なるほど。画像のパッチ分割（ViT）と似てるな」

娘：「近い。でも音声トークンには特徴がある。**RVQ（Residual Vector Quantization）**で階層的に量子化するから、1秒の音声が**複数層×時間ステップの2次元トークン行列**になる。例えばMimiなら**32kbpsで8層×50Hz=毎秒400トークン**」

父：「階層ってどういうことだ？」

娘：「1層目が大まかな構造（ざっくりした音色）、2層目以降が細かい詳細（ノイズや高域成分）。**層を増やすほど高音質になるが、トークン数も増える**。トレードオフを調整できるのがRVQの強み」

父：「つまり、音声トークンは『音声のレゴブロック』で、ブロックの数（層数）を増やすとより精密な音声が組み立てられる」

娘：「大正解😊 **MoshiやPersonaPlex、Seeduplex**など最新の音声LLMは全て音声トークンを前提に設計されてる。トークンなしには始まらない——**音声AIの共通部品**だよ」
